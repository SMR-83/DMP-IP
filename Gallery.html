REDCap Project Setup

Here is what the main page of REDCap looks like to begin customizing up a project

Data Collection Instruments

Instruments to capture primary and secondary data. I structured the data collection instruments according to the study protocol. Primary data is collected daily, secondary data is collected weekly, and pretests and post-tests on the quality of life and bowel function surveys. There is also an instrument to collect minimal patient data to support the subgroup analysis mentioned in the protocol.

Data Collection Fields

The data collection fields contained in the data collection instruments. I used the protocol to annotate the data collection fields with relevant information. You can also see the dropdown menu of responses that I set up to standardize the data that is captured.

SF-36 and FBDSI Mock-up surveys

To get an approximate idea about how many KB this data would require, I made a mock-up of a 36 question quality of life survey. The mock-up of the FBDSI survey contains question more closely related to the actual survey.

Edit Checks on Integer Limits

Here you can see how I set integer limits for a data collection field. There is also an example of the warning message that appears if a user enters a value outside of the limits

Edit Checks on Email

Here you can see the participant intake fields from the user's point of view. I set an edit check on the email entry field, so if a user inputs something other than an email address, they are prompted with an error message.

Required Fields

Fields that are required are marked with a red asterisk and note that says "must provide value". I used this feature for most of the data collection in mock-up patient diary. In this example, sex is a required field because it relates to the subgroup analysis mentioned in the study protocol.

Identifying Information Options

Similar to the way data collection fields can be marked as required, there is another option to mark data collection fields as containing information that identifies a participant. Here you can see the de-identification options available when exporting data.
